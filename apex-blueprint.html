10 Killer Features To Build

🏦
Black-Litterman Signal Fusion
APEX Signature Feature

Use TFT quantile forecasts (q10/q50/q90) as Bayesian "views" into the Black-Litterman model. The model blends your predictions with CAPM equilibrium returns, automatically scaling confidence by forecast uncertainty. Replaces the static TFT×0.5 weight.

🌡️
Dynamic Bayesian Ensemble Weights
APEX Signature Feature

Instead of fixed weights (TFT=0.5, RSI=0.2...), APEX tracks the prediction accuracy of each alpha over a rolling 500-bar window and adjusts weights via Bayesian updating. Signals that perform better get more weight automatically.

📡
Reddit Sentiment → Ensemble
AWET (finally wired)

AWET already collects Reddit mention counts and sentiment scores but never uses them. APEX wires these into the ensemble as a 10% weight signal. High sentiment divergence from price = contrarian signal. Low divergence = momentum confirmation.

📈
Stochastic + ATR as New Alphas
From Lean

Stochastic %K/%D catches overbought/oversold states that RSI misses (RSI is momentum, Stochastic is mean-reversion relative to range). ATR-based position sizing ensures larger positions in low-volatility regimes, smaller in high-volatility. Lean provides both as warmup-aware indicators.

🎯
VWAP-Based Execution
From Lean

Current system places immediate market orders — worst execution possible. Lean's VolumeWeightedAveragePriceExecutionModel splits orders across VWAP windows, reducing market impact by ~3–8 bps per trade. At 5 orders/min, this compounds into significant alpha preservation.

🌐
ETF Universe Selection
From Lean

Replace universe.csv with Lean's ETFConstituentsUniverseSelectionModel("SPY") + liquidity filter. Automatically trades the 100 most liquid S&P 500 stocks, updated daily. Avoids penny stocks, low-volume names, and recently delisted securities.

🤖
Signal Analyst LLM Agent
NEW APEX Feature

A new agent powered by qwen2.5:32b that runs after every ensemble decision and produces a 3-sentence human-readable explanation: why the signal was generated, which indicators agreed/disagreed, and what the primary risk is. Logs to audit_trail for every signal.

🔄
Auto Walk-Forward Reviewer
NEW APEX Feature

After each nightly training run, a Backtest Reviewer Agent uses the LLM to analyse fold results, flag any fold where OOS Sharpe is an outlier (>2σ from median), and detect potential leakage. Produces a PDF report and pushes a summary to Telegram.

💾
TimescaleDB Continuous Aggregates
NEW APEX Feature

Use TimescaleDB's continuous aggregate views to pre-compute OHLCV candles at 5m/15m/1h resolutions from 1m raw data. Feature engineering becomes a SELECT on a materialized view instead of a full table scan. 50–200× faster signal computation at scale.

📦
Signal Provider API
NEW APEX Feature

Expose a FastAPI /signals endpoint: GET /signals/{symbol} returns the latest scored signal, confidence, regime, and recommended position size. This turns APEX into a standalone signal provider — other systems or dashboards can consume it without touching the Kafka pipeline.

Key Code Patterns to Implement

Using TFT quantile forecasts as Black-Litterman "views". The Q50 forecast = expected return, (Q90-Q10) = uncertainty → scales tau automatically.
# apex/portfolio/black_litterman.py import numpy as np class TFTBlackLittermanModel: """Use TFT quantile forecasts as Bayesian views into Black-Litterman.""" def __init__(self, risk_aversion: float = 2.5, tau: float = 0.05): self.risk_aversion = risk_aversion self.tau = tau # uncertainty in equilibrium returns def compute_targets( self, signals: list[ScoredSignal], cov_matrix: np.ndarray, market_caps: np.ndarray, ) -> dict[str, float]: n = len(signals) # Equilibrium returns (CAPM prior) pi = self.risk_aversion * cov_matrix @ market_caps pi = pi / pi.sum() # Build view matrix P (one row per signal with calibrated confidence) P = np.zeros((n, n)) Q = np.zeros(n) # view returns = TFT q50 forecasts omega = np.zeros((n, n)) # view uncertainty for i, sig in enumerate(signals): P[i, i] = 1.0 Q[i] = sig.horizon_30_q50 # TFT median forecast as view # Uncertainty = width of prediction interval (q90-q10) # Wider interval = less confident view interval = sig.horizon_30_q90 - sig.horizon_30_q10 omega[i, i] = (interval / 4) ** 2 # ~95% CI assumption # Black-Litterman posterior tau_cov = self.tau * cov_matrix M = np.linalg.inv( np.linalg.inv(tau_cov) + P.T @ np.linalg.inv(omega) @ P ) posterior_returns = M @ ( np.linalg.inv(tau_cov) @ pi + P.T @ np.linalg.inv(omega) @ Q ) # Scale by calibrated confidence from Platt scaler weights = { sig.symbol: float(posterior_returns[i] * sig.calibrated_confidence) for i, sig in enumerate(signals) } # Normalize to sum = 1 (long-only for paper trading) total = sum(w for w in weights.values() if w > 0) return {k: max(0, v/total) for k, v in weights.items()}
Replace static weights (TFT=0.5, RSI=0.2...) with a Bayesian tracker that updates weights based on recent prediction accuracy per alpha source.
# apex/signal_engine/bayesian_weights.py class BayesianAlphaWeighter: """Tracks prediction accuracy per alpha source over rolling window. Updates weights using Bayesian beta-binomial model. """ def __init__(self, sources: list[str], window: int = 500, prior: float = 0.5): self.window = window # Beta distribution parameters (successes, failures) per source self.alpha = {s: prior * 10 for s in sources} self.beta_ = {s: prior * 10 for s in sources} self._history = {s: [] for s in sources} def update(self, source: str, was_correct: bool): # Rolling window: remove oldest observation hist = self._history[source] if len(hist) >= self.window: oldest = hist.pop(0) if oldest: self.alpha[source] = max(1, self.alpha[source] - 1) else: self.beta_[source] = max(1, self.beta_[source] - 1) hist.append(was_correct) if was_correct: self.alpha[source] += 1 else: self.beta_[source] += 1 def get_weights(self) -> dict[str, float]: # Posterior mean = alpha / (alpha + beta) raw = {s: self.alpha[s] / (self.alpha[s] + self.beta_[s]) for s in self.alpha} total = sum(raw.values()) return {s: v / total for s, v in raw.items()} def get_effective_accuracy(self, source: str) -> float: return self.alpha[source] / (self.alpha[source] + self.beta_[source])

""" DEFAULT_ETF = "SPY" MIN_AVG_VOLUME = 500_000 # shares/day (Unified's liquidity filter) MIN_PRICE = 5.0 MIN_MARKET_CAP = 1_000_000_000 TOP_N = 100 async def select(self, date: datetime) -> list[str]: # Step 1: Get ETF constituents (Polygon reference data) constituents = await self._fetch_etf_constituents(self.DEFAULT_ETF, date) # Step 2: Apply filters filtered = [] for symbol in constituents: stats = await self._fetch_stats(symbol, date, lookback_days=10) if (stats["avg_volume"] >= self.MIN_AVG_VOLUME and stats["price"] >= self.MIN_PRICE and stats["market_cap"] >= self.MIN_MARKET_CAP and not self._near_earnings(symbol, date)): filtered.append((symbol, stats["avg_volume"])) # Step 3: Return top N by liquidity filtered.sort(key=lambda x: x[1], reverse=True) selected = [s for s, _ in filtered[:self.TOP_N]] # Step 4: Publish to universe.selected Kafka topic await self.kafka.publish("universe.selected", { "date": date.isoformat(), "symbols": selected, "count": len(selected), "etf_source": self.DEFAULT_ETF, }) return selected

Replace immediate market orders with VWAP-based execution to minimize slippage. Adapted from Lean's VolumeWeightedAveragePriceExecutionModel.
# apex/execution/vwap_executor.py class VWAPExecutionModel: """ Splits target order into slices based on volume profile. Goal: achieve VWAP or better on entry/exit. Adapted from Lean's VolumeWeightedAveragePriceExecutionModel. """ MAX_ORDER_PCT_OF_VOLUME = 0.10 # max 10% of bar volume per slice SLICE_INTERVAL_SECONDS = 60 async def execute( self, symbol: str, target_shares: int, direction: str, correlation_id: str, ): filled = 0 slices = [] while filled < target_shares: bar = await self._get_current_bar(symbol) vwap = bar.vwap current_price = bar.close # Only execute if price is favorable vs VWAP if direction == "UP" and current_price > vwap * 1.002: await asyncio.sleep(self.SLICE_INTERVAL_SECONDS) continue # wait for better price # Slice = min(remaining, max_pct * bar_volume) slice_qty = min( target_shares - filled, int(bar.volume * self.MAX_ORDER_PCT_OF_VOLUME) ) if slice_qty < 1: await asyncio.sleep(self.SLICE_INTERVAL_SECONDS) continue order = await self.alpaca.submit_limit_order( symbol=symbol, qty=slice_qty, side=direction.lower(), limit_price=round(current_price * 1.001, 2), time_in_force="ioc" # immediate-or-cancel ) filled += order.filled_qty slices.append(order) await asyncio.sleep(self.SLICE_INTERVAL_SECONDS) return { "symbol": symbol, "total_filled": filled, "slices": len(slices), "correlation_id": correlation_id, "avg_fill_price": sum(s.avg_price * s.filled_qty for s in slices) / filled }
